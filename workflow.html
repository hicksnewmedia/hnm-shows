What triggers a new deploy

Any git push to main

Editing a show page, adding a video, updating stats, changing any HTML — one push kicks off the full deploy automatically.

Show Manager export + push

Download an updated show file from the Show Manager, drop it in the right subfolder, then push — same auto-deploy kicks off.

Update Workflow

How to Update the Site

There are two ways to update show content. Use whichever fits the change you're making.

Both paths end the same way No matter how you edit — Show Manager or direct file edit — you always finish with a git push. That's the only action that triggers a live deploy.

Method A — Show Manager

Best for: videos, stats, host info, any content update

1
Open the Show Manager Go to shows.hicksnewmedia.com/show-manager.html in any browser — your changes auto-save there
2
Pick a show & make edits Update videos, analytics stats, or host bios in the Videos / Analytics / Hosts tabs
3
Export → Download the file Click the Export tab, then the download button for the show you edited
4
Rename to index.html Replace the file in the matching show subfolder on your Mac:
team-no-sleep-index.html → shows/team-no-sleep/index.html
tek-forum-index.html → shows/tek-forum/index.html
digital-collective-index.html → shows/digital-collective/index.html
5
Open Terminal, navigate to the repo Run these three commands:
cd "~/Documents/Claude/Projects/Show Pages/shows"
git add . && git commit -m "update show content"
git push
Live in ~30 seconds GitHub Actions handles the rest automatically

Method B — Direct File Edit

Best for: layout tweaks, copy edits, adding a new show

1
Open the file in a text editor Navigate to Show Pages/shows/ on your Mac and open the HTML file for the page you want to edit
2
Make your changes Edit the HTML directly — copy, images, links, layout, whatever you need
3
Save the file ⌘S in your editor
4
Open Terminal and push
cd "~/Documents/Claude/Projects/Show Pages/shows"
git add . && git commit -m "edit description here"
git push
Live in ~30 seconds GitHub Actions handles the rest automatically
Page structure Each show lives in its own subfolder. The hub page is shows/index.html.
